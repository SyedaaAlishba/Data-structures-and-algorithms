# Doubly Linked List Implementation (C++)

## Overview

This project implements a **Doubly Linked List** using **C++**. It demonstrates how elements can be dynamically stored in memory while allowing traversal in both forward and backward directions.

A **Doubly Linked List** is a type of linked list in which each node contains a reference to both the **next node** and the **previous node**. This structure allows efficient insertion, deletion, and traversal from either direction.

The program performs fundamental operations such as inserting elements, displaying the list, searching for a value, and deleting nodes.

---

## Features

* Insert elements into the doubly linked list
* Display all elements in the list
* Search for a specific value
* Delete nodes from the list
* Dynamic memory management
* Bidirectional traversal capability

---

## What is a Doubly Linked List?

A **Doubly Linked List** is a linear data structure where each node contains:

* **Data** – the stored value
* **Next Pointer** – reference to the next node
* **Previous Pointer** – reference to the previous node

Unlike singly linked lists, this structure allows movement **in both directions**, making certain operations more efficient.

## Real World Applications

Doubly linked lists are widely used in systems where navigation in both directions is required.

### 1. Web Browser Navigation

Browsers use structures similar to doubly linked lists for **Back and Forward navigation** between pages.

### 2. Music or Video Playlists

Media players allow users to move to the **next or previous track**, which can be implemented using a doubly linked list.

### 3. Undo and Redo Operations

Software like text editors and graphic design tools manage **undo and redo history** using structures similar to doubly linked lists.

### 4. Navigation Systems

Applications that allow moving forward and backward between steps or pages can utilize doubly linked lists.

### 5. Memory Management

Operating systems sometimes use linked structures to track memory allocation and deallocation.
